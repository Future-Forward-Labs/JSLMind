# Phase 4 Design — Hybrid RAG on JSL Docs

**Date:** 2026-03-08
**Status:** Approved
**Pillar:** 4 — Hybrid RAG on JSL Documents
**Timeline:** Day 4 (Blueprint Section 7)

---

## Problem Statement

JSL plant managers and analysts need to query SOPs, grade specifications, and maintenance manuals in natural language and get answers with precise source citations. The pipeline must support live document ingestion (drag-and-drop) and sophisticated hybrid retrieval that outperforms naive keyword or semantic search alone.

---

## Architecture

### Ingestion Path (write)

```
/rag/docs/corpus/ (local dir, Docker volume)
      │ poll every 5s (Camel file:// component)
      ▼
FileToQdrantRoute.java (Camel SpringBoot)
• Detects new PDF / Word / Excel files
• POSTs file path to FastAPI POST /ingest
      │
      ▼
rag-service: POST /ingest
• Unstructured.io → chunks + metadata (page, section, doc_type, grade)
• BGE-M3 via Ollama → dense embeddings
• Qdrant upsert (collection: jsl_docs)
• BM25 index update (persisted: rag/bm25_index.pkl)
```

### Retrieval Path (read)

```
Kong AI Gateway  (dept API key)
      │ POST /rag/query
      ▼
rag-service: POST /rag/query
• Qdrant dense search (BGE-M3, top-20)
• BM25 sparse search (top-20)
• Metadata filter (grade, doc_type) — optional
• RRF fusion: score(d) = Σ 1/(k + rank_i(d)), k=60
• Re-ranked top-8 chunks → LiteLLM (Claude / Mistral)
• Response: answer + citations [{doc, section, page, score, chunk}]
```

## API Contract

### POST /ingest (internal — called by Camel)

```json
// Request
{ "file_path": "/rag/docs/corpus/Grade_316L_Specification.pdf" }

// Response
{ "doc_id": "uuid", "chunks": 24, "status": "indexed" }
```

### POST /rag/query (public — through Kong)

```json
// Request
{
  "query": "Max carbon content for Grade 316L?",
  "filters": { "grade": "316L", "doc_type": "spec" },
  "top_k": 5
}

// Response
{
  "answer": "The maximum carbon content for Grade 316L is 0.03%...",
  "sources": [
    {
      "doc": "Grade_316L_Specification.pdf",
      "section": "3.2 Chemical Composition",
      "page": 4,
      "score": 0.91,
      "chunk": "Carbon (C): max 0.03%..."
    }
  ],
  "retrieval_debug": {
    "dense_hits": 20,
    "sparse_hits": 20,
    "rrf_merged": 8
  }
}
```

### Qdrant chunk metadata payload

```json
{
  "doc_name": "Grade_316L_Specification.pdf",
  "doc_type": "spec",
  "grade": "316L",
  "section": "3.2 Chemical Composition",
  "page": 4,
  "text": "Carbon (C): max 0.03%..."
}
```

## Integration Changes

### Camel (integration/)

- `SharepointToQdrantRoute.java` → renamed/repurposed as `FileToQdrantRoute.java`
- Uses `file://` component polling `/rag/docs/corpus` every 5s
- On new file: HTTP POST to `http://rag-service:8001/ingest`

### Docker Compose (docker-compose.yml)

- Add `rag-service` container — ports 8001 (ingest), 8002 (query)
- Volume: `./rag/docs/corpus:/rag/docs/corpus`
- Add `unstructured` container (official image)
- Qdrant already present — collection `jsl_docs` created on rag-service startup

### Kong (gateway/kong/kong.yaml)

- Add upstream: `rag-service` (port 8002)
- Add route: `/rag/query` → `rag-service`
- Apply existing dept key auth plugin

### Backstage (catalog/)

- Add `catalog/data-products/rag-pipeline.yaml` (kind: Component, type: service)
